在Mac上安装Python的经过

在Mac上面，我安装了Homebrew，Anaconda和Xcode。我通过Homebrew手动安装了Python3。Anaconda默认会安装Python3，而Xcode则会默认安装Python2和Python3。

在一开始，它们给我带来了很多困扰，不知道自己现在用的是哪个版本的Python。我安装Xcode，是为了解决安装其他软件的依赖性，一般不会使用它。通过Homebrew安装的Python3，只能在命令行使用，学习成本较高。所以在刚开始学习Python的时候，我就依赖于Anaconda了。这样我不用考虑那些安装细节，只专注在Python和编程本身。

我用Anaconda熟悉了Python的语法，并且完成了一些小的脚本之后，就准备做一些稍微复杂的项目。这个时候，我就想搞明白如何来管理Python的包和虚拟环境。因此就开始研究这些细节。

为了简化研究的复杂度，我先卸载了通过Homebrew安装的Python。这样，系统内只剩下Xcode和Anaconda安装的Python了。

where python

/usr/bin/python
/usr/local/anaconda3/bin/python

where python3

/usr/bin/python3
/usr/local/anaconda3/bin/python3

这样我就知道了，上面的路径是Xcode的安装路径，下边的则是Anaconda的安装路径。

随后，我再通过Homebrew安装了Python。

brew install python

安装完毕之后，我将/usr/local/bin放在/usr/bin之前。但是发现系统首选的Python还是Xcode安装的版本。

这里要注意一个小坑。我摘录了下来：

Virtualenv mapping caveat

  • Pipenv automatically maps projects to their specific virtualenvs.
  • The virtualenv is stored globally with the name of the project’s root directory plus the hash of the full path to the project’s root (e.g., my_project-a3de50).
  • If you change your project’s path, you break such a default mapping and pipenv will no longer be able to find and to use the project’s virtualenv.
  • You might want to set export PIPENV_VENV_IN_PROJECT=1 in your .bashrc/.zshrc (or any shell configuration file) for creating the virtualenv inside your project’s directory, avoiding problems with subsequent path changes.

我按照上面的方法做了，因为我可能会更改项目文档的存放路径。

str()与repr()的区别

两个函数都能够输出字符串，粗看之下并无区别，如下面的例子。

print(str('Hello World!'))
Hello World!
print(repr('Hello World!'))
'Hello World!'
print(str(2.0 / 11.0))
0.18181818181818182
print(repr(2.0 / 11.0))
0.18181818181818182

除了打印字符串有引号的区别，看不出来有啥不同。

换一个例子，就能够看出它们的区别来。

print(str(today))
2021-08-13 21:23:12.463664
print(repr(today))
datetime.datetime(2021, 8, 13, 21, 23, 12, 463664)

str()的输出很容易阅读，即便是用户也能看明白。而repr()的输出则没有那么直观，但是能够告诉编程人员关于对象的信息，如上述例子中的类型和属性。

它们的区别主要是：

  • str()是为了可读性设计的，repr()是为了明确性设计的。前者方便用户阅读，后者用于开发和调试。
  • str()的信息是用于打印，repr()提供了对象的详细信息。
  • str()调用的是对象的__str_函数，repr()调用的是对象的__repr__函数。如果要修改自定义类的输出，直接修改自定义类的__str_和__repr__函数即可。

在虚拟环境中使用IPython

我启用了虚拟环境后，在其中安装了IPython，但是运行的时候却发现，执行的并非是虚拟环境的，还是之前Anaconda中的。系统也会给出警告信息。

ipython

/usr/local/anaconda3/lib/python3.8/site-packages/IPython/core/interactiveshell.py:937: UserWarning: Attempting to work in a virtualenv. If you encounter problems, please install IPython inside the virtualenv.
  warn("Attempting to work in a virtualenv. If you encounter problems, please "
Python 3.8.11 (default, Aug  6 2021, 08:56:27)
Type 'copyright', 'credits' or 'license' for more information
IPython 7.26.0 -- An enhanced Interactive Python. Type '?' for help.

搜索之下，我才知道，这与Bash的缓存有关。之前我已经执行过其它版本的IPython。因此，如果想要执行虚拟环境内的版本，需要先清除缓存。也有文章讲直接重新打开Bash也可以，不过我没有在自己使用的Zsh上面验证过。如果要清除所有的Bash缓存，执行：

hash -r

如果只是清除其中某一个命令，执行：

hash -d ipython

我们也可以使用type命令来查看可执行程序的完整路径。

type ipython

ipython is /Users/zhoulinux/Test_Py/.venv/bin/ipython

在网络上，我还看到另外一个解决方案，但是我没有明白是如何解决的。先记录下来吧。

alias ipy="python -c 'import IPython; IPython.terminal.ipapp.launch_new_instance()'"

IPython在虚拟环境中安装完毕后，只能够在命令行中使用。相比而言，我更喜欢使用Jupyter QtConsole。

pip install qtconsole
# Make sure that Qt is installed. Unfortunately, Qt is not installed when using pip.
pip install pyqt5
