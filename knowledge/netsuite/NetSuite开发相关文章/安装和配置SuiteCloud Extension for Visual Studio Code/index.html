安装插件

按步骤安装完插件后，系统提示Java版本不匹配。

Your Java version 12.0.1 is not compatible with SuiteCloud Extension for Visual Studio Code. Install Oracle JDK version 11.

我之前是使用Brew安装的Java，不想因为这个就改变了原来的版本，因此打算再安装一个Java11，然后把VSCode的Java路径配置为Java11。

先安装Java11。

~ brew install openjdk@11

执行结果如下所示。

==> Caveats
For the system Java wrappers to find this JDK, symlink it with
  sudo ln -sfn /usr/local/opt/openjdk@11/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-11.jdk

openjdk@11 is keg-only, which means it was not symlinked into /usr/local,
because this is an alternate version of another formula.

If you need to have openjdk@11 first in your PATH, run:
  echo 'export PATH="/usr/local/opt/openjdk@11/bin:$PATH"' >> ~/.zshrc

For compilers to find openjdk@11 you may need to set:
  export CPPFLAGS="-I/usr/local/opt/openjdk@11/include"

根据安装信息，来检验是否成功。

~ /usr/local/opt/openjdk@11/bin/java --version
openjdk 11.0.12 2021-07-20
OpenJDK Runtime Environment Homebrew (build 11.0.12+0)
OpenJDK 64-Bit Server VM Homebrew (build 11.0.12+0, mixed mode)

我接下来去修改VSCode中的Java配置，但是却无法找到java.home这个配置项，让我非常疑惑。后来在网上搜索的时候，发现可能是没有安装Java相关插件Extension Pack for Java。安装完毕后，我就能够找到这个配置项，并做了修改。

"java.home": /usr/local/opt/openjdk@11/bin
"java.home": /usr/local/opt/openjdk@11
"java.home": /usr/local/opt/openjdk@11/libexec/openjdk.jdk

上面几个都不行，均无法在Java Language Server这个选项内找到我安装的Java11。

重启了几次VSCode之后，我推测可能是系统无法检测到，又回头查看了安装提示信息，发现可能需要建立链接才可以。

我就按照上面的提示信息建立了链接。

~ sudo ln -sfn /usr/local/opt/openjdk@11/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-11.jdk

做完上述修改后，继续修改了java.home的值。

"java.home": "/Library/Java/JavaVirtualMachines/openjdk-11.jdk/Contents/Home"

启动VSCode时，依然提示Java版本为12。但是此时，能够在Java Language Server这个选项内找到我安装的Java11了。

我推测，可能需要按照提示信息修改环境变量。

~ echo 'export PATH="/usr/local/opt/openjdk@11/bin:$PATH"' >> ~/.zshrc

修改完毕后，再打开VSCode的时候，就没有SuiteCloud插件就没有提示Java版本的错误了，而且自动下载了SuiteCloud SDK。之前在执行新建SDF项目的命令时，插件还提示需要下载SuiteCloud SDK。

设置账号

有两种方式，一种是通过浏览器，另一种是直接使用TBA。

我本来想使用TBA的方式来进行验证，但是没有成功，一直提示错误。死磕之后，我只能尝试通过浏览器来进行验证，很顺利就成功了。结果发现，它的底层也是通过TBA来实现的，不过更便捷。

这件事情，让我意识到了自己经常犯的一个错误，就是有完美主义倾向。像上面的两种验证方式，我在尝试浏览器验证之后，就可以直接去做脚本开发了。相对来说，脚本开发才是我现阶段最需要掌握的内容。TBA验证的方式，我完全可以先放下来，等到有时间和经验之后，再来解决它。或者，我直接找一名开发同事，帮忙看一下就结束了，不用在哪里死磕而浪费时间。我不用为了证明自己的能力，而一定要解决当前遇到的每个问题。有的时候，只要这个问题不影响主计划的执行，我就可以把它放在一边，而去完成最重要的事情。那点心理上的吹毛求疵，完全可以不用理会。

这对于我来讲，是一个挑战。好一点的是，昨天晚上意识到之后，我就强迫自己不再关注它，而是把它这个问题记录下来。希望这是一个好的开始，在之后能够做的更好。

上边提到的，把遇到的问题记录下来，是可以帮助我去除这种完美主义心理的。相当于，我把这个问题从大脑当中暂时清理出来，放在一边，让大脑能够去做更重要的事情。同时，这个问题也不会被我遗忘，而是在时机更恰当的时候进行处理。这样做，能够极大地缓解我的焦虑心理。
